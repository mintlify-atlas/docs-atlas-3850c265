---
title: Db
description: The main database interface for SlateDB
---

The `Db` struct represents a SlateDB database instance. It provides methods for reading and writing key-value data, managing transactions, and controlling database lifecycle.

## Opening a Database

### open()

Open a new database with default options.

```rust
pub async fn open<P: Into<Path>>(
    path: P,
    object_store: Arc<dyn ObjectStore>,
) -> Result<Db, Error>
```

<ParamField path="path" type="P: Into<Path>" required>
  The path to the database
</ParamField>

<ParamField path="object_store" type="Arc<dyn ObjectStore>" required>
  The object store to use for the database
</ParamField>

<ResponseField name="Result" type="Result<Db, Error>">
  Returns the database instance on success, or an error if the database could not be opened
</ResponseField>

**Example:**

```rust
use slatedb::{Db, Error};
use slatedb::object_store::memory::InMemory;
use std::sync::Arc;

#[tokio::main]
async fn main() -> Result<(), Error> {
    let object_store = Arc::new(InMemory::new());
    let db = Db::open("test_db", object_store).await?;
    Ok(())
}
```

## Read Operations

### get()

Get a value from the database with default read options.

```rust
pub async fn get<K: AsRef<[u8]> + Send>(
    &self,
    key: K
) -> Result<Option<Bytes>, Error>
```

<ParamField path="key" type="K: AsRef<[u8]> + Send" required>
  The key to get
</ParamField>

<ResponseField name="Result" type="Result<Option<Bytes>, Error>">
  Returns `Some(Bytes)` if the key exists, `None` if it doesn't exist, or an error
</ResponseField>

<Note>
The `Bytes` object returned contains a slice of an entire 4 KiB block. The block will be held in memory as long as the caller holds a reference to the `Bytes` object. Consider copying the data if you need to hold it for a long time.
</Note>

**Example:**

```rust
use slatedb::{Db, Error};
use slatedb::object_store::{ObjectStore, memory::InMemory};
use std::sync::Arc;

#[tokio::main]
async fn main() -> Result<(), Error> {
    let object_store: Arc<dyn ObjectStore> = Arc::new(InMemory::new());
    let db = Db::open("test_db", object_store).await?;
    db.put(b"key", b"value").await?;
    assert_eq!(db.get(b"key").await?, Some("value".into()));
    Ok(())
}
```

### scan()

Scan a range of keys using the default scan options.

```rust
pub async fn scan<K, T>(&self, range: T) -> Result<DbIterator, Error>
where
    K: AsRef<[u8]> + Send,
    T: RangeBounds<K> + Send
```

<ParamField path="range" type="T: RangeBounds<K> + Send" required>
  The range of keys to scan (e.g., `"a".."z"`, `..`, `"key"..`)
</ParamField>

<ResponseField name="Result" type="Result<DbIterator, Error>">
  Returns an iterator with the results of the scan
</ResponseField>

**Example:**

```rust
use slatedb::{Db, Error};
use slatedb::object_store::{ObjectStore, memory::InMemory};
use std::sync::Arc;

#[tokio::main]
async fn main() -> Result<(), Error> {
    let object_store: Arc<dyn ObjectStore> = Arc::new(InMemory::new());
    let db = Db::open("test_db", object_store).await?;
    db.put(b"a", b"a_value").await?;
    db.put(b"b", b"b_value").await?;

    let mut iter = db.scan("a".."b").await?;
    let kv = iter.next().await?.unwrap();
    assert_eq!(kv.key.as_ref(), b"a");
    assert_eq!(kv.value.as_ref(), b"a_value");
    assert_eq!(None, iter.next().await?);
    Ok(())
}
```

### merge()

Merge a value into the database with default `MergeOptions` and `WriteOptions`.

Merge operations allow applications to bypass the traditional read/modify/write cycle by expressing partial updates using an associative operator. The merge operator must be configured when opening the database.

```rust
pub async fn merge<K, V>(&self, key: K, value: V) -> Result<WriteHandle, Error>
where
    K: AsRef<[u8]>,
    V: AsRef<[u8]>
```

<ParamField path="key" type="K: AsRef<[u8]>" required>
  The key to merge into
</ParamField>

<ParamField path="value" type="V: AsRef<[u8]>" required>
  The merge operand to apply
</ParamField>

<ResponseField name="Result" type="Result<WriteHandle, Error>">
  Returns a write handle containing the sequence number and timestamp of the merge operation, or an error if no merge operator is configured
</ResponseField>

**Example:**

```rust
use slatedb::{Db, Error, MergeOperator, MergeOperatorError};
use slatedb::object_store::{ObjectStore, memory::InMemory};
use std::sync::Arc;
use bytes::Bytes;

struct StringConcatMergeOperator;

impl MergeOperator for StringConcatMergeOperator {
    fn merge(&self, _key: &Bytes, existing_value: Option<Bytes>, value: Bytes) -> Result<Bytes, MergeOperatorError> {
        let mut result = existing_value.unwrap_or_default().as_ref().to_vec();
        result.extend_from_slice(&value);
        Ok(Bytes::from(result))
    }
}

#[tokio::main]
async fn main() -> Result<(), Error> {
    let object_store: Arc<dyn ObjectStore> = Arc::new(InMemory::new());
    let db = Db::builder("test_db", object_store)
        .with_merge_operator(Arc::new(StringConcatMergeOperator))
        .build()
        .await?;
    let handle = db.merge(b"key", b"value").await?;
    Ok(())
}
```

## Flush Operations

### flush()

Flush in-memory writes to disk. This function blocks until the in-memory data has been durably written to object storage.

If WAL is enabled, this method is equivalent to:
`flush_with_options(FlushOptions { flush_type: FlushType::Wal })`

If WAL is disabled, this method is equivalent to:
`flush_with_options(FlushOptions { flush_type: FlushType::Memtable })`

```rust
pub async fn flush(&self) -> Result<(), Error>
```

<ResponseField name="Result" type="Result<(), Error>">
  Returns `Ok(())` if the flush was successful, or an error
</ResponseField>

**Example:**

```rust

use slatedb::object_store::{ObjectStore, memory::InMemory};
use std::sync::Arc;

#[tokio::main]

    let object_store: Arc<dyn ObjectStore> = Arc::new(InMemory::new());

    Ok(())
```

## Snapshot Operations

### snapshot()

Create a snapshot of the database. The snapshot represents a consistent view of the database at the time it is created.

```rust
pub async fn snapshot(&self) -> Result<Arc<DbSnapshot>, Error>
```

<ResponseField name="Result" type="Result<Arc<DbSnapshot>, Error>">
  Returns a snapshot of the database, which provides read-only access to the database state at the time of creation
</ResponseField>

**Example:**

```rust
use slatedb::{Db, Error};
use slatedb::object_store::{ObjectStore, memory::InMemory};
use std::sync::Arc;
use bytes::Bytes;

#[tokio::main]
async fn main() -> Result<(), Error> {
    let object_store = Arc::new(InMemory::new());
    let db = Db::open("test_db", object_store).await?;

    // Write some data and create a snapshot
    db.put(b"key1", b"value1").await?;
    let snapshot = db.snapshot().await?;

    // Snapshot provides read-only access to database state
    let value = snapshot.get(b"key1").await?;
    assert_eq!(value, Some(Bytes::from(b"value1".as_ref())));

    // Write more data to original database
    db.put(b"key2", b"value2").await?;

    // Snapshot still sees old state, original db sees new data
    assert_eq!(snapshot.get(b"key2").await?, None);
    assert_eq!(db.get(b"key2").await?, Some(Bytes::from(b"value2".as_ref())));

    Ok(())
}
```

### status()

Check the database status.

This is a passive check that does not perform any I/O. The status is checked at least every `Settings::manifest_poll_interval`, but might also be updated based on other internal events or user-facing operations.

Once a database is closed, either normally or due to an error, it can't be reopened. A new `Db` instance must be created to access the database again.

```rust
pub fn status(&self) -> Result<(), Error>
```

<ResponseField name="Result" type="Result<(), Error>">
  - `Ok(())` if the DB is still open
  - `Err(ErrorKind::Closed)` if the DB was closed normally
  - `Err(e)` if the DB was closed with an error, where `e` is the error that caused the closure
</ResponseField>

### subscribe()

Subscribe to database state changes.

Returns a `tokio::sync::watch::Receiver<DbStatus>` that always reflects the latest database status. For example, you can wait for a specific sequence number to become durable.

```rust
pub fn subscribe(&self) -> tokio::sync::watch::Receiver<DbStatus>
```

<ResponseField name="Receiver" type="tokio::sync::watch::Receiver<DbStatus>">
  Returns a receiver that can be used to watch database status changes
</ResponseField>

<Warning>
**Deadlock risk:** The returned receiver holds a read lock on the current value while borrowed. The database must acquire a write lock to publish new status updates. Holding the read guard for an extended period will block all database status updates and may cause a deadlock. Always clone or copy the data you need.
</Warning>
