## Overview

Checkpoints in SlateDB provide persistent, recoverable snapshots of the database state. Unlike in-memory snapshots, checkpoints are stored in the manifest and can be used for backup, recovery, and creating new database instances from a specific point in time.

## Types

### Checkpoint

```rust
#[non_exhaustive]
#[derive(Clone, PartialEq, Serialize, Debug)]
pub struct Checkpoint {
    pub id: Uuid,
    pub manifest_id: u64,
    pub expire_time: Option<DateTime<Utc>>,
    pub create_time: DateTime<Utc>,
    pub name: Option<String>,
}
```

Represents a checkpoint with metadata.

**Fields:**
- `id` - Unique identifier for the checkpoint
- `manifest_id` - The manifest version this checkpoint references
- `expire_time` - Optional expiration time for automatic cleanup
- `create_time` - When the checkpoint was created
- `name` - Optional human-readable name for the checkpoint

### CheckpointCreateResult

```rust
#[non_exhaustive]
#[derive(Debug, Clone)]
pub struct CheckpointCreateResult {
    pub id: Uuid,
    pub manifest_id: u64,
}
```

Result returned when creating a checkpoint.

**Fields:**
- `id` - The ID of the created checkpoint
- `manifest_id` - The manifest ID referenced by the checkpoint

## Methods

### create_checkpoint

```rust
pub async fn create_checkpoint(
    &self,
    scope: CheckpointScope,
    options: &CheckpointOptions
) -> Result<CheckpointCreateResult, crate::Error>
```

Creates a checkpoint of an opened database using the provided options.

**Parameters:**
- `scope` - The scope of the checkpoint (`CheckpointScope::All` to flush all data)
- `options` - Checkpoint configuration options

**Returns:**
- `Result<CheckpointCreateResult, crate::Error>` - The ID and manifest ID of the created checkpoint

**Example:**

```rust
use slatedb::{Db, CheckpointScope};
use slatedb::config::CheckpointOptions;

let db = Db::open("/tmp/my_db", object_store).await?;

// Write some data
db.put(b"key1", b"value1").await?;
db.put(b"key2", b"value2").await?;

// Create a checkpoint with all data flushed
let result = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions::default()
).await?;

println!("Created checkpoint {} at manifest {}", result.id, result.manifest_id);
```

## CheckpointOptions

```rust
pub struct CheckpointOptions {
    pub lifetime: Option<Duration>,
    pub source: Option<Uuid>,
    pub name: Option<String>,
}
```

**Fields:**
- `lifetime` - Optional duration after which the checkpoint expires
- `source` - Optional source checkpoint ID to create a checkpoint from another checkpoint
- `name` - Optional human-readable name for the checkpoint

## CheckpointScope

```rust
pub enum CheckpointScope {
    All,
}
```

**Variants:**
- `All` - Flush all data (WAL and memtables) before creating the checkpoint

## Usage Examples

### Basic Checkpoint Creation

```rust
use slatedb::{Db, CheckpointScope};
use slatedb::config::CheckpointOptions;

let db = Db::open("/tmp/my_db", object_store).await?;

db.put(b"key1", b"value1").await?;

// Create a checkpoint
let checkpoint = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions::default()
).await?;

println!("Checkpoint ID: {}", checkpoint.id);
```

### Checkpoint with Expiration

```rust
use std::time::Duration;
use slatedb::config::CheckpointOptions;

// Create a checkpoint that expires in 1 hour
let checkpoint = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions {
        lifetime: Some(Duration::from_secs(3600)),
        ..Default::default()
    }
).await?;
```

### Named Checkpoint

```rust
use slatedb::config::CheckpointOptions;

// Create a named checkpoint for easy identification
let checkpoint = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions {
        name: Some("before_migration".to_string()),
        ..Default::default()
    }
).await?;
```

### Checkpoint from Checkpoint

```rust
use slatedb::config::CheckpointOptions;

// Create an initial checkpoint
let checkpoint1 = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions::default()
).await?;

// Create another checkpoint from the first one
let checkpoint2 = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions {
        source: Some(checkpoint1.id),
        ..Default::default()
    }
).await?;

// checkpoint2 references the same manifest as checkpoint1
assert_eq!(checkpoint1.manifest_id, checkpoint2.manifest_id);
```

## Admin Operations

Checkpoint management operations are available through the `Admin` API:

### List Checkpoints

```rust
use slatedb::admin::AdminBuilder;

let admin = AdminBuilder::new("/tmp/my_db", object_store).build();

// List all checkpoints
let all_checkpoints = admin.list_checkpoints(None).await?;

// List checkpoints by name
let named_checkpoints = admin.list_checkpoints(Some("backup")).await?;
```

### Refresh Checkpoint

Extend the lifetime of a checkpoint:

```rust
use std::time::Duration;

// Extend checkpoint lifetime by 24 hours
admin.refresh_checkpoint(
    checkpoint_id,
    Some(Duration::from_secs(86400))
).await?;
```

### Delete Checkpoint

```rust
// Delete a checkpoint by ID
admin.delete_checkpoint(checkpoint_id).await?;
```

### Create Detached Checkpoint

Create a checkpoint without an open database:

```rust
use slatedb::admin::AdminBuilder;
use slatedb::config::CheckpointOptions;

let admin = AdminBuilder::new("/tmp/my_db", object_store).build();

let checkpoint = admin.create_detached_checkpoint(
    &CheckpointOptions::default()
).await?;
```

## Use Cases

### Backup and Recovery

```rust
// Create a daily backup checkpoint
let backup = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions {
        name: Some(format!("daily_backup_{}", chrono::Utc::now().format("%Y%m%d"))),
        lifetime: Some(Duration::from_secs(30 * 86400)), // 30 days
        ..Default::default()
    }
).await?;

println!("Backup created with ID: {}", backup.id);
```

### Pre-Migration Checkpoints

```rust
// Create a checkpoint before a risky operation
let pre_migration = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions {
        name: Some("pre_schema_migration".to_string()),
        ..Default::default()
    }
).await?;

// Perform migration
migrate_schema(&db).await?;

// If successful, can delete the checkpoint
admin.delete_checkpoint(pre_migration.id).await?;
```

### Testing and Development

```rust
// Create a checkpoint with test data
let test_data = db.create_checkpoint(

        ..Default::default()
    }

```

## Checkpoint Lifecycle

1. **Creation** - Checkpoint is created with `create_checkpoint()` or `create_detached_checkpoint()`
2. **Storage** - Checkpoint metadata is stored in the manifest
3. **Usage** - Checkpoint can be listed, refreshed, or used for recovery
4. **Expiration** - If a lifetime is set, checkpoint expires automatically
5. **Deletion** - Checkpoint can be manually deleted with `delete_checkpoint()`

## Behavior with Flushes

When using `CheckpointScope::All`, the checkpoint operation will:

1. Flush all WAL entries (if WAL is enabled)
2. Flush all memtables to SST files
3. Create a checkpoint referencing the resulting manifest

This ensures that all data is durably written to object storage before the checkpoint is created.

```rust
let db = Db::open("/tmp/my_db", object_store).await?;

// Write data to memtable
db.put(b"key1", b"value1").await?;

// Create checkpoint - this will trigger flush
let checkpoint = db.create_checkpoint(
    CheckpointScope::All,
    &CheckpointOptions::default()
).await?;

// At this point, all data is flushed to SST files
```
