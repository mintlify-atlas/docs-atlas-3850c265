## Overview

`DbSnapshot` provides a consistent, point-in-time view of the database. It allows you to read data as it existed at the moment the snapshot was created, isolated from any subsequent writes or modifications.

## Structure

```rust
pub struct DbSnapshot {
    txn_id: Uuid,
    started_seq: u64,
    txn_manager: Arc<TransactionManager>,
    db_inner: Arc<DbInner>,
}
```

### Fields

- `txn_id` - The ID of the transaction that created this snapshot
- `started_seq` - The sequence number when the snapshot was created
- `txn_manager` - Reference to the transaction manager
- `db_inner` - Reference to the database

## Methods

### get

```rust
pub async fn get<K: AsRef<[u8]> + Send>(
    &self,
    key: K
) -> Result<Option<Bytes>, crate::Error>
```

Retrieves a value from the snapshot using default read options.

**Parameters:**
- `key` - The key to retrieve

**Returns:**
- `Result<Option<Bytes>, crate::Error>` - The value if it exists, `None` otherwise

**Example:**

```rust
use slatedb::Db;

let db = Db::open("/tmp/my_db", object_store).await?;

// Write some data
db.put(b"key1", b"value1").await?;

// Create a snapshot
let snapshot = db.snapshot().await?;

// Modifications after snapshot creation are not visible
db.put(b"key1", b"modified").await?;

// Snapshot still sees original value
let value = snapshot.get(b"key1").await?;
assert_eq!(value, Some(Bytes::from("value1")));
```

### scan

```rust
pub async fn scan<K, T>(&self, range: T) -> Result<DbIterator, crate::Error>
where
    K: AsRef<[u8]> + Send,
    T: RangeBounds<K> + Send,
```

Scans a range of keys using default scan options.

**Parameters:**
- `range` - The range of keys to scan

**Returns:**
- `Result<DbIterator, crate::Error>` - An iterator with the scan results

**Example:**

```rust
let db = Db::open("/tmp/my_db", object_store).await?;

db.put(b"key1", b"value1").await?;
db.put(b"key2", b"value2").await?;
db.put(b"key3", b"value3").await?;

let snapshot = db.snapshot().await?;

// Write more data after snapshot
db.put(b"key4", b"value4").await?;

// Snapshot scan only sees keys 1-3
let mut iter = snapshot.scan(b"key1"..).await?;
while let Some(kv) = iter.next().await? {
    println!("{:?}: {:?}", kv.key, kv.value);
}
```

## Traits

### DbRead

`DbSnapshot` implements the `DbRead` trait, providing a consistent read interface.

### Drop

When a `DbSnapshot` is dropped, it automatically unregisters from the transaction manager, freeing up resources.

## Usage Notes

### Isolation

Snapshots provide isolation from subsequent writes:

```rust
let db = Db::open("/tmp/my_db", object_store).await?;

db.put(b"key1", b"original").await?;

// Create snapshot at this point
let snapshot = db.snapshot().await?;

// Modify the database
db.put(b"key1", b"modified").await?;
db.delete(b"key2").await?;

// Snapshot sees the original state
let value = snapshot.get(b"key1").await?;
assert_eq!(value, Some(Bytes::from("original")));
```

### Multiple Snapshots

You can create multiple snapshots at different points in time:

```rust
db.put(b"key1", b"version1").await?;
let snapshot1 = db.snapshot().await?;

db.put(b"key1", b"version2").await?;
let snapshot2 = db.snapshot().await?;

db.put(b"key1", b"version3").await?;
let snapshot3 = db.snapshot().await?;

// Each snapshot sees its respective version
assert_eq!(snapshot1.get(b"key1").await?, Some(Bytes::from("version1")));
assert_eq!(snapshot2.get(b"key1").await?, Some(Bytes::from("version2")));
assert_eq!(snapshot3.get(b"key1").await?, Some(Bytes::from("version3")));
```

### Snapshots and Flushes

Snapshots remain consistent even when data is flushed to disk:

```rust
db.put(b"key1", b"value1").await?;
let snapshot = db.snapshot().await?;

// Flush data and write new values
db.flush().await?;
db.put(b"key1", b"modified").await?;

// Snapshot still sees original value
let value = snapshot.get(b"key1").await?;
assert_eq!(value, Some(Bytes::from("value1")));
```
