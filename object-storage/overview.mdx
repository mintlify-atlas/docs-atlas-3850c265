## How Object Storage Works with SlateDB

SlateDB uses object storage to persist:

- **SST files**: Sorted string tables containing your data
- **WAL segments**: Write-ahead log for durability
- **Manifest files**: Metadata about database state and structure

```rust
use slatedb::Db;
use std::sync::Arc;

// Create an object store (example using in-memory)
let object_store = Arc::new(object_store::memory::InMemory::new());

// Open SlateDB with the object store
let db = Db::open("/tmp/slatedb", object_store).await?;

// Use the database
db.put(b"key", b"value").await?;
let value = db.get(b"key").await?;

db.close().await?;
```

## Choosing an Object Storage Backend

Consider these factors when selecting an object storage backend:

### Performance
- **Latency**: Choose a region close to your compute resources
- **Throughput**: Ensure the service can handle your workload's read/write volume
- **Consistency**: SlateDB requires strong consistency guarantees

### Cost
- **Storage costs**: Price per GB stored
- **Request costs**: Price per GET/PUT operation
- **Data transfer**: Egress charges for data leaving the cloud

### Operational Considerations
- **Authentication**: How you manage credentials and access control
- **Availability**: SLA and redundancy requirements
- **Compliance**: Data residency and regulatory requirements

<Warning>
  SlateDB requires object stores that support conditional PUT operations (like ETag matching) to ensure consistency. Make sure your chosen backend supports this feature.
</Warning>

## Environment Variables

Most object storage backends can be configured using environment variables, which is useful for production deployments:

```bash
# AWS S3
export AWS_REGION=us-east-1
export AWS_ACCESS_KEY_ID=your-access-key
export AWS_SECRET_ACCESS_KEY=your-secret-key
export AWS_BUCKET_NAME=your-bucket

# Azure
export AZURE_STORAGE_ACCOUNT_NAME=your-account
export AZURE_STORAGE_ACCESS_KEY=your-key
export AZURE_STORAGE_CONTAINER_NAME=your-container

# Google Cloud Storage
export GOOGLE_SERVICE_ACCOUNT=path/to/credentials.json
export GCS_BUCKET_NAME=your-bucket
```
