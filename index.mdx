## Why SlateDB?

SlateDB is an embedded storage engine that leverages object storage (S3, GCS, Azure, MinIO) as its primary storage layer. Unlike traditional LSM-tree engines that write to local disk, SlateDB writes directly to object storage, providing:

- **Bottomless storage capacity** — No need to provision disk space
- **High durability** — Built-in replication and redundancy from object storage
- **Easy cloning and replication** — Copy databases by copying object prefixes
- **Cost-effective storage** — Pay for what you use with object storage pricing

## Quick Example

Get started with SlateDB in your Rust project:

```rust
use slatedb::{Db, Error};
use slatedb::object_store::{ObjectStore, memory::InMemory};
use std::sync::Arc;

#[tokio::main]
async fn main() -> Result<(), Error> {
    // Setup
    let object_store: Arc<dyn ObjectStore> = Arc::new(InMemory::new());
    let db = Db::open("/tmp/my_database", object_store).await?;

    // Put
    db.put(b"hello", b"world").await?;

    // Get
    let value = db.get(b"hello").await?;
    assert_eq!(value, Some("world".into()));

    // Delete
    db.delete(b"hello").await?;

    Ok(())
}
```
